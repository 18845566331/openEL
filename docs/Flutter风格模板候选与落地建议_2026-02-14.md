# Flutter 风格模板候选与落地建议（2026-02-14）

## 结论
Flutter 有可直接复用的风格模板与官方样例，建议优先从官方样例起步，再按桌面平台补充设计语言组件库。

## 推荐候选（按优先级）
1. 官方样例仓库 `flutter/samples`
   - 适合：快速搭建可维护骨架、导航结构、桌面交互。
   - 建议重点参考：
     - `material_3_demo`（现代 Material 3 组件体系）
     - `desktop_photo_search`（桌面端 Material + Fluent UI 组合思路）
     - `navigation_and_routing`（`go_router` 路由方案）

2. Material 3 Demo（官方）
   - 适合：作为 B2（纸质工作流风格）与通用管理后台的基础视觉。
   - 优点：官方维护，和 Flutter Material 组件演进同步。

3. `fluent_ui`（Windows 风格组件库）
   - 适合：C2（专业中台）在 Windows 的原生观感实现。
   - 说明：社区维护，适合 Windows 端部署项目。

4. `macos_ui`（macOS 风格组件库）
   - 适合：如后续需要 macOS 原生风格版本。
   - 说明：主要面向 macOS。

## 与当前三套 GUI 映射
1. A2 工业命令中心（`YYvep`）
   - 推荐基座：官方 `flutter/samples` + 自定义深色主题 + `NavigationRail`
2. B2 纸质报告工作流（`opk5Y`）
   - 推荐基座：`material_3_demo` + 卡片化表单流程 + 导出流程页
3. C2 专业中台（`Qwzo5`）
   - 推荐基座：`desktop_photo_search` 思路 + `fluent_ui`（Windows）

## 落地步骤（建议）
1. 先用官方样例确定导航与页面骨架（顶部菜单、左栏、中图、右栏）。
2. 再把当前 Pencil 的 A2/B2/C2 视觉映射到 Flutter `ThemeData`（颜色、圆角、字体、间距）。
3. 最后接入现有后端 ONNX 推理接口，补齐批量检测、标注、统计、导出链路。

## 当前环境限制
本机当前终端未识别 `flutter` 命令（SDK 未加入 PATH 或未安装完整工具链），需要先完成 Flutter SDK 安装与环境变量配置后再执行模板拉取与运行。
