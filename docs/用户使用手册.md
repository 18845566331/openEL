## 1. 系统概述

EL光伏组件缺陷检测系统是一款用于光伏组件EL（电致发光）图像缺陷检测的桌面软件。系统能够自动识别EL图像中的隐裂、断栅、黑斑、烧结异常等常见缺陷，并提供可视化标注和统计报告。

### 主要功能

- **模型配置**：加载和配置ONNX格式的缺陷检测模型
- **单张检测**：对单张EL图像进行缺陷检测，快速查看结果
- **批量检测**：对整个目录的EL图像进行批量检测，自动统计
- **结果查看**：查看检测结果列表、缺陷统计图表和可视化图像
- **报告导出**：将批量检测结果导出为CSV格式报告

### 界面布局

系统界面采用三栏布局：

- **左侧导航栏**：点击图标切换功能模块（模型配置、单张检测、批量检测、结果查看）
- **中央工作区**：显示当前功能模块的操作界面
- **右侧数据面板**：显示系统状态（模型是否已加载、当前阈值参数）和最近一次检测的统计数据

## 2. 系统启动

### 一键启动（推荐）

1. 找到系统根目录下的 **`一键启动系统.bat`** 文件
2. 双击该文件
3. 系统将自动启动后端服务和前端界面
4. 等待前端窗口出现即可开始使用

### 分步启动

如果需要单独启动后端或前端，可以使用以下脚本：

- **启动后端服务**：双击 **`启动后端服务.bat`**，等待控制台显示服务地址 `http://127.0.0.1:5000`
- **启动前端界面**：双击 **`启动前端界面.bat`**，等待桌面窗口出现

> 注意：前端界面需要后端服务正常运行才能使用检测功能。请确保先启动后端服务。

## 3. 模型配置

在使用检测功能之前，需要先加载ONNX缺陷检测模型。点击左侧导航栏的 **"模型配置"** 图标进入配置页面。

### 方式一：手动配置加载

1. **选择模型文件**：点击"ONNX模型路径"右侧的 **"浏览"** 按钮，在弹出的文件选择对话框中选择 `.onnx` 模型文件
2. **填写缺陷类别标签**：在"缺陷类别标签"输入框中填写模型对应的类别名称，多个类别用英文逗号分隔（例如：`隐裂,断栅,黑斑,烧结异常`）
3. **设置输入尺寸**：填写模型要求的输入宽度和高度，通常为 `640 × 640`
4. **选择输出布局**：根据模型的输出格式选择对应的布局选项：
   - `cxcywh_obj_cls`：标准 Anchor-Based 格式（最常用）
   - `xyxy_score_class`：左上右下坐标格式
   - `cxcywh_score_class`：中心点坐标格式
5. **选择推理引擎**：
   - `onnxruntime`：推荐，性能更好，支持GPU加速
   - `opencv`：备选引擎
6. **配置预处理参数**：
   - **归一化**：通常保持开启
   - **颜色通道交换**：通常保持开启
7. **调节阈值参数**：拖动滑块设置置信度阈值和IOU阈值（详见[参数调节说明](#8-参数调节说明)）
8. 点击 **"加载模型"** 按钮

加载成功后，页面顶部会显示绿色提示"模型加载成功"，右侧数据面板的"模型状态"会变为"已加载"。

### 方式二：通过配置文件加载

如果已有JSON格式的模型配置文件，可以一键加载：

1. 点击"配置文件路径"右侧的 **"浏览"** 按钮，选择 `.json` 配置文件
2. 点击 **"通过配置文件加载"** 按钮

配置文件模板位于 `backend/config/model_profile.example.json`，可复制后修改使用。

### 恢复默认参数

点击 **"恢复默认"** 按钮可将所有参数重置为默认值（置信度阈值 0.55、IOU阈值 0.45 等）。

## 4. 单张图像检测

点击左侧导航栏的 **"单张检测"** 图标进入单张检测页面。

### 操作步骤

1. **选择图像文件**：点击 **"浏览"** 按钮，在弹出的对话框中选择一张EL图像文件
   - 支持的格式：JPG、JPEG、PNG、BMP、TIF、TIFF
2. **调节检测参数**（可选）：
   - 拖动滑块调节置信度阈值和IOU阈值
   - 开启"保存可视化结果"开关（如需保存带标注的结果图像）
3. 点击 **"开始检测"** 按钮
4. 等待检测完成（页面顶部会显示进度条）

### 查看结果

检测完成后，页面下方会显示：

- **检测结果**：OK（无缺陷）或 NG（有缺陷）
- **缺陷数量**：检测到的缺陷总数
- **缺陷明细**：每个缺陷的类别名称、置信度分数和位置坐标
- **可视化图像**：如果开启了保存可视化，会显示带有检测框标注的图像

右侧数据面板也会同步更新检测统计信息。

---

## 5. 批量图像检测

点击左侧导航栏的 **"批量检测"** 图标进入批量检测页面。

### 操作步骤

1. **选择图像目录**：点击 **"浏览"** 按钮，选择包含EL图像的文件夹
2. **配置扫描选项**：
   - **递归扫描子目录**：开启后会扫描所有子文件夹中的图像
   - **文件扩展名过滤**：指定要处理的图像格式，默认为 `jpg,jpeg,png,bmp,tif,tiff`
   - **最大图像数量**：拖动滑块设置本次最多处理的图像数量（100 ~ 5000）
3. **调节检测参数**（可选）：
   - 拖动滑块调节置信度阈值和IOU阈值
   - 开启"保存可视化结果"开关（如需为每张NG图像保存标注图像）
4. 点击 **"开始批量检测"** 按钮
5. 等待检测完成（页面顶部会显示进度条）

### 查看结果

检测完成后，页面下方会显示：

- **批量统计**：总图像数、OK数量、NG数量、总缺陷数
- **缺陷分类统计**：每种缺陷类别的数量和占比进度条
- **图像列表**：显示前20条检测结果，每条包含文件名和检测结果（OK/NG）

右侧数据面板也会同步更新完整的统计信息。

## 6. 结果查看和可视化

点击左侧导航栏的 **"结果查看"** 图标进入结果展示页面。

### 页面布局

- **左侧区域**：显示统计卡片（总图像/OK/NG数量）和缺陷分布图表
- **右侧区域**：显示检测结果列表

### 查看详情

在结果列表中点击任意一行，可以展开查看该图像的详细信息：

- 每个缺陷的类别名称、置信度分数和边界框坐标
- 可视化图像的保存路径（如果有）

### 统计图表

页面左侧会显示：

- **OK/NG比例图**：直观展示合格与不合格的比例
- **缺陷类别分布图**：展示各类缺陷的数量分布

## 7. 报告导出

在 **"结果查看"** 页面，当有批量检测结果时，页面右上角会显示 **"导出CSV报告"** 按钮。

### 导出步骤

1. 确保已完成一次批量检测（结果查看页面有数据显示）
2. 点击页面右上角的 **"导出CSV报告"** 按钮
3. 在弹出的保存对话框中选择文件保存位置和文件名
4. 点击保存，等待导出完成
5. 导出成功后会显示绿色提示，包含文件保存路径

### CSV报告内容

导出的CSV文件包含以下结构：

**项目信息头部：**

| 行 | 内容 |
|------|------|
| 项目名称 | 项目名称 |
| 报告编号 | 报告编号 |
| 委托单位 | 委托单位名称 |
| 检测单位 | 检测单位名称 |
| 检测人员 | 检测人员姓名 |
| 签发日期 | 签发日期 |

**缺陷统计：**

| 缺陷类别 | 数量 |
|------|------|
| 隐裂 | N |
| 断栅 | N |

**文件检测结果：**

| 列名 | 说明 |
|------|------|
| 文件名 | 图像文件名 |
| 检测结果 | OK（无缺陷）或 NG（有缺陷） |
| 文件路径 | 图像文件的完整路径 |

CSV文件使用UTF-8-BOM编码，可直接用Excel打开并正确显示中文。

## 8. 参数调节说明

系统提供两个关键参数供用户调节，以优化不同场景下的检测效果。

### 置信度阈值

- **含义**：模型对检测结果的"确信程度"。只有置信度高于此阈值的检测结果才会被保留。
- **范围**：0.00 ~ 1.00
- **默认值**：0.55
- **调节建议**：
  - **提高阈值**（如 0.7 ~ 0.9）：减少误检（假阳性），但可能漏检一些不明显的缺陷
  - **降低阈值**（如 0.3 ~ 0.5）：减少漏检，但可能增加误检
  - **推荐范围**：0.45 ~ 0.65，根据实际检测效果微调

### IOU阈值

- **含义**：用于去除重叠检测框的参数。当两个检测框的重叠面积比（交并比）超过此阈值时，只保留置信度更高的那个。
- **范围**：0.00 ~ 1.00
- **默认值**：0.45
- **调节建议**：
  - **提高阈值**（如 0.6 ~ 0.8）：允许更多重叠框保留，适用于缺陷密集的场景
  - **降低阈值**（如 0.2 ~ 0.4）：更积极地去除重叠框，适用于缺陷稀疏的场景
  - **推荐范围**：0.35 ~ 0.55，一般无需频繁调整

### 调节方法

在模型配置、单张检测、批量检测页面中，均可通过拖动滑块调节这两个参数。滑块右侧会实时显示当前数值。

## 9. 常见问题和错误解决方法

### 启动相关

**问题：双击 .bat 文件后窗口一闪而过**

- 原因：可能是Python或Flutter环境未正确安装
- 解决：右键点击 .bat 文件，选择"在终端中打开"或"用命令提示符运行"，查看具体错误信息

**问题：后端服务启动失败，提示 `ModuleNotFoundError`**

- 原因：Python依赖包未安装
- 解决：打开命令提示符，进入 `backend` 目录，执行以下命令安装依赖：
  ```
  cd backend
  python -m venv venv
  venv\Scripts\activate
  pip install -r requirements.txt
  ```

**问题：前端界面无法启动，提示找不到 `flutter` 命令**

- 原因：Flutter SDK未安装或未添加到系统环境变量
- 解决：安装Flutter SDK后，将 `flutter\bin` 目录添加到系统 `PATH` 环境变量中，然后重新启动

### 连接相关

**问题：前端界面提示"网络连接失败"或无法连接后端**

- 原因：后端服务未启动或端口被占用
- 解决：
  1. 确认后端服务已启动（控制台应显示 `http://127.0.0.1:5000`）
  2. 在浏览器中访问 `http://127.0.0.1:5000/health` 检查服务是否正常
  3. 检查防火墙是否阻止了5000端口
  4. 如果端口被占用，关闭占用端口的程序后重新启动

### 模型相关

**问题：模型加载失败**

- 原因：模型文件路径错误、文件损坏或格式不兼容
- 解决：
  1. 确认模型文件路径正确，文件确实存在
  2. 确认模型文件为有效的 `.onnx` 格式
  3. 检查模型文件是否完整（未被截断或损坏）
  4. 尝试切换推理引擎（从 `onnxruntime` 切换到 `opencv`，或反之）

**问题：检测时提示"模型未就绪"**

- 原因：尚未加载模型
- 解决：先切换到"模型配置"页面，加载模型后再进行检测

**问题：GPU加速不生效，推理速度较慢**

- 原因：未安装GPU版本的推理引擎或CUDA环境
- 解决：
  1. 确认电脑有NVIDIA显卡并已安装CUDA驱动
  2. 安装GPU版本的ONNX Runtime：`pip install onnxruntime-gpu`
  3. 系统会自动检测GPU并启用加速

### 检测相关

**问题：检测结果全部为OK，但图像明显有缺陷**

- 原因：置信度阈值设置过高
- 解决：降低置信度阈值（例如从0.55降到0.35），重新检测

**问题：检测结果有大量误检（标注了不是缺陷的区域）**

- 原因：置信度阈值设置过低
- 解决：提高置信度阈值（例如从0.55提高到0.70），重新检测

**问题：同一个缺陷被标注了多个重叠的框**

- 原因：IOU阈值设置过高
- 解决：降低IOU阈值（例如从0.45降到0.30），使系统更积极地去除重叠框

**问题：批量检测中部分图像检测失败**

- 原因：个别图像文件损坏或格式不支持
- 解决：系统会自动跳过失败的图像并继续处理其余图像。失败的图像会在结果中标记错误信息，可根据提示检查对应文件

### 报告导出相关

**问题：CSV报告在Excel中打开显示乱码**

- 原因：Excel未正确识别文件编码
- 解决：
  1. 系统默认使用UTF-8-BOM编码，大多数情况下Excel可正确打开
  2. 如仍有乱码，在Excel中使用"数据 → 从文本/CSV"功能导入，手动选择UTF-8编码

**问题：点击"导出CSV报告"按钮无反应**

- 原因：当前没有批量检测结果
- 解决：先执行一次批量检测，有结果后再导出报告

### 其他问题

**问题：界面显示异常或控件错位**

- 解决：尝试调整窗口大小，或关闭后重新启动前端界面

**问题：检测速度很慢**

- 解决：
  1. 启用GPU加速（参见上方GPU相关说明）
  2. 适当减小模型输入尺寸（如从640改为320，但可能影响检测精度）
  3. 批量检测时适当减少最大图像数量
