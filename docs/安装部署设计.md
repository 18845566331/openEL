# 安装部署设计（Windows）

## 1. 环境前提

1. 操作系统：Windows 10/11 64 位
2. Python：3.10+
3. Flutter：3.22+（用于桌面端）
4. 模型：已训练好的本地 ONNX 文件

## 2. 后端部署

项目根目录执行：

```powershell
cd el_defect_system
powershell -ExecutionPolicy Bypass -File .\scripts\setup_backend.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\start_backend.ps1
```

服务健康检查：

```powershell
curl http://127.0.0.1:5000/health
```

## 3. 前端部署

```powershell
cd el_defect_system\frontend
flutter pub get
flutter run -d windows
```

## 4. 生产打包建议

### 后端

1. 使用 `Nuitka` 或 `PyInstaller` 打包为单目录
2. 把 `backend/config/*.json` 与模型配置作为外部文件
3. 服务注册为 Windows 服务（NSSM 或 Task Scheduler）

### 前端

1. 执行 `flutter build windows --release`
2. 发布 `build/windows/x64/runner/Release/`
3. `exe` 与后端服务同机部署或局域网部署

## 5. 闭源商用合规设计

1. 新程序不携带特定训练框架源码。
2. 推理仅依赖 ONNX Runtime / OpenCV DNN（宽松许可证）。
3. 发布包需保留第三方 LICENSE 与版权声明文档。
4. 模型来源与授权边界需在合同中明确。

## 6. 现场部署参数建议

1. `confidence_threshold`：0.50 - 0.70
2. `iou_threshold`：0.40 - 0.55
3. 若产线误检偏高，优先提高 `confidence_threshold`
4. 若漏检偏高，优先降低 `confidence_threshold`

## 7. 运维检查清单

1. 后端服务自动重启策略是否启用
2. 模型与配置版本号是否记录
3. 日志目录是否做容量上限
4. 每次模型更新是否留存回归结果
