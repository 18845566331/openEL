1. 系统简介
------------------------------------------------------------------------

EL 光伏组件缺陷检测系统是一款专业的光伏组件 EL（电致发光）图像
缺陷检测软件。系统采用深度学习 ONNX 模型进行智能检测，能够自动
识别 EL 图像中的隐裂、断栅、黑斑、烧结异常等常见缺陷。

主要特点：
  • 支持 ONNX 格式检测模型，兼容多种训练框架导出的模型
  • 支持 ONNX Runtime 和 OpenCV DNN 双推理引擎
  • 支持 NVIDIA GPU (CUDA) 加速推理
  • 单张图像检测和批量目录检测
  • 检测结果可视化（在原图上绘制检测框和标签）
  • 检测报告导出（CSV 和 Word 格式）
  • 图片分割模式，自动识别并裁剪电池片区域
  • 全中文桌面 GUI 界面

2. 快速入门
------------------------------------------------------------------------

步骤一：启动程序
  双击桌面快捷方式「EL 光伏组件缺陷检测系统」
  或从开始菜单找到「EL 光伏组件缺陷检测系统」启动

步骤二：选择检测模式
  1. 登录系统后进入工作台页面
  2. 通过左侧导航栏切换检测模式
  3. 系统会自动向服务器申请对应的检测模型
  4. 模型加载完成后即可开始检测
  注意：无需手动加载模型文件，模式切换后系统自动完成

步骤三：执行检测
  单张检测：
    1. 点击左侧导航栏的「单张检测」图标
    2. 选择一张 EL 图像文件
    3. 点击「开始检测」

  批量检测：
    1. 点击左侧导航栏的「批量检测」图标
    2. 选择包含 EL 图像的文件夹
    3. 配置扫描选项（递归扫描、文件类型、最大数量等）
    4. 点击「开始批量检测」

步骤四：查看结果与导出报告
  1. 点击左侧导航栏的「结果查看」图标
  2. 查看检测统计和结果列表
  3. 点击「导出报告」按钮导出 CSV 或 Word 格式报告

3. 功能说明
------------------------------------------------------------------------

3.1 检测模式与模型管理
  • 自动模型加载：切换检测模式时系统自动向服务器申请模型
  • 推理引擎：支持 ONNX Runtime（推荐）和 OpenCV DNN
  • 参数调节：置信度阈值和 IOU 阈值实时可调
  • 无需手动配置模型文件路径或类别标签

3.2 单张检测
  • 支持格式：JPG、JPEG、PNG、BMP、TIF、TIFF
  • 可视化：可选择保存带检测框标注的结果图像
  • 结果展示：显示缺陷类别、置信度和位置信息

3.3 批量检测
  • 目录扫描：支持递归扫描子目录
  • 文件过滤：可指定处理的图像格式
  • 数量限制：可设置最大处理图像数量（100~5000）
  • 统计汇总：自动统计 OK/NG 数量和缺陷分布

3.4 图片分割模式
  • 自动识别 EL 图像中的电池片/组件区域
  • 将识别出的区域裁剪为独立子图保存
  • 支持透视裁剪、自动裁剪、忽略边缘等功能
  • 可自定义输出质量、分辨率和保存路径

3.5 报告导出
  • CSV 报告：包含项目信息、缺陷统计和文件检测结果
  • Word 报告：基于专业模板生成检测报告文档
  • 编码：CSV 文件使用 UTF-8-BOM 编码，支持中文 Excel 直接打开

4. 参数调节指南
------------------------------------------------------------------------

4.1 检测参数

  ● 置信度阈值（Confidence Threshold）
  ──────────────────────────────────────
    含义：模型对检测结果的「确信程度」，仅输出高于此阈值
          的检测结果
    范围：0.00 ~ 1.00
    默认：0.55

    调节建议：
    • 数值越高，检测结果越精确，但可能漏掉不太明显的缺陷
    • 数值越低，检测越灵敏，但可能出现误检
    • 提高到 0.7~0.9：严格筛选，减少误检
    • 降低到 0.3~0.5：灵敏检测，减少漏检
    • 推荐范围：0.45 ~ 0.65

  ● IOU 阈值（IOU Threshold）
  ──────────────────────────────────────
    含义：去除重叠检测框的参数。IOU（交并比）表示两个
          检测框的重叠面积占合并面积的比例
    范围：0.00 ~ 1.00
    默认：0.45

    调节建议：
    • 数值越高，保留更多重叠框，适用于缺陷密集场景
    • 数值越低，更积极地去除重叠框，适用于缺陷稀疏场景
    • 推荐范围：0.35 ~ 0.55

  ● CUDA 开关
  ──────────────────────────────────────
    含义：启用 NVIDIA GPU 加速推理
    默认：开启

    说明：
    • 开启后使用 GPU 进行推理计算，大幅提升检测速度
    • 需要电脑安装 NVIDIA 显卡及驱动
    • 若无 NVIDIA 显卡，即使开启也会自动回退到 CPU 推理
    • 批量检测时建议保持开启以提升效率

  ● FP16 开关（半精度浮点）
  ──────────────────────────────────────
    含义：使用 16 位浮点精度代替 32 位进行推理计算
    默认：开启

    说明：
    • 开启后显存占用降低约 50%，推理速度提升 20%~50%
    • 精度损失极小，几乎不影响检测准确率
    • 仅在启用 CUDA（GPU 加速）时生效
    • 建议保持开启，除非遇到特殊兼容性问题

  ● NMS 开关（非极大值抑制）
  ──────────────────────────────────────
    含义：自动去除同一缺陷区域的重复检测框
    默认：关闭

    说明：
    • 开启后对检测结果进行后处理，去除高度重叠的多余框
    • 当同一缺陷区域出现多个检测框时建议开启
    • 去除标准由上方的 IOU 阈值控制
    • 部分模型已内置 NMS，此时无需额外开启

4.2 分割设置参数（图片分割模式专用）

  切换到「图片分割」模式后，工作台左侧会出现「分割设置」面板。

  ── 检测选项 ──

  ● 忽略画面边缘
  ──────────────────────────────────────
    含义：过滤触碰到图像边缘的组件区域
    用途：EL 图像边缘常有不完整的电池片，启用后自动排除
          这些不完整区域，避免产生无效的裁剪子图
    建议：批量处理时建议开启

  ● 透视裁剪
  ──────────────────────────────────────
    含义：对倾斜拍摄的 EL 图像进行透视校正后再裁剪
    用途：当 EL 图像拍摄角度不正时，启用此选项可自动
          矫正透视变形，输出正视角度的裁剪子图
    建议：图像有明显倾斜或梯形变形时开启

  ● 自动裁剪
  ──────────────────────────────────────
    含义：分割识别后自动裁剪保存，无需手动点击「全部裁剪」
    用途：启用后检测完成时自动将每个电池片区域裁剪导出
    建议：批量处理时开启可全自动完成分割+裁剪

  ── 输出参数 ──

  ● 外扩（Expand Pixels）
  ──────────────────────────────────────
    含义：裁剪区域向外扩展的像素数
    范围：0 ~ 100 px
    默认：0

    说明：
    • 设为 0 时严格按检测边界裁剪
    • 设为 10~20 时在裁剪区域四周各扩展对应像素
    • 适用于需要保留组件周围少量背景的场景

  ● 质量（Quality）
  ──────────────────────────────────────
    含义：裁剪子图的图像压缩质量
    范围：30% ~ 100%
    默认：95%

    说明：
    • 100% = 无损质量，文件较大
    • 95% = 视觉无损，推荐用于后续分析和报告
    • 70%~80% = 适中质量，适合大批量处理节省存储空间
    • 低于 50% 可能影响图像清晰度

  ● 分辨率（Resolution）
  ──────────────────────────────────────
    含义：裁剪子图的输出尺寸（宽 × 高）
    单位：像素 (px)
    默认：留空（保持原始裁剪大小）

    说明：
    • 留空时裁剪子图保持检测区域的原始像素尺寸
    • 填写具体数值（如 640×640）后所有子图统一缩放到该尺寸
    • 适用于需要统一尺寸输入到下游模型训练的场景

  ── 保存路径 ──

  ● 保存路径
  ──────────────────────────────────────
    含义：裁剪子图的保存目录
    默认：与源图同目录

    说明：
    • 留空时裁剪结果保存在源图所在目录
    • 可点击「选择」按钮指定自定义保存路径
    • 批量裁剪时会自动保持源文件的目录层级结构

5. 常见问题 (FAQ)
------------------------------------------------------------------------

Q: 启动程序后界面未显示？
A: 请等待 5~10 秒，后端服务需要时间启动。如仍未显示，请检查
   是否有杀毒软件阻止了程序运行。

Q: 切换检测模式后模型未加载？
A: 请检查：
   1. 网络连接是否正常，服务器是否可达
   2. 是否已登录授权服务器
   3. 当前账号是否有权使用该检测模式
   4. 等待几秒后重试，模型下载可能需要时间

Q: 检测结果全部为 OK，但明显有缺陷？
A: 置信度阈值设置过高，请降低至 0.35 左右重新检测。

Q: 检测结果有大量误检？
A: 置信度阈值设置过低，请提高至 0.70 左右重新检测。

Q: CSV 报告在 Excel 中显示乱码？
A: 系统默认使用 UTF-8-BOM 编码。如仍有问题，在 Excel 中使用
   「数据 → 从文本/CSV」功能导入，手动选择 UTF-8 编码。

Q: GPU 加速如何启用？
A: 在检测参数面板中开启 CUDA 开关。需要安装 NVIDIA GPU 驱动，
   系统会自动检测并使用 GPU 加速。

Q: 批量检测中部分图像失败？
A: 系统会自动跳过失败图像继续处理。失败图像会在结果中标记错误
   信息，请检查对应文件是否损坏或格式不支持。

Q: 分割裁剪的子图不完整或有黑边？
A: 请调整以下参数：
   1. 增加「外扩」值（10~20px），避免裁剪过于贴近边界
   2. 开启「忽略画面边缘」，排除不完整的边缘组件
   3. 若图片倾斜，开启「透视裁剪」矫正变形
