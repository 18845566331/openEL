# 技术实现说明（EL 光伏组件缺陷检测）

## 1. 架构总览

系统采用前后端分离：

1. `frontend`（Flutter 桌面 GUI）
2. `backend`（本地推理 API）

数据流：

1. 前端选择 ONNX 模型与图像/目录
2. 调用后端 `load_model` 加载模型
3. 调用 `detect` 或 `detect/batch` 获取缺陷结果
4. 前端显示统计并输出报告

## 2. 推理核心（无特定训练框架依赖）

后端引擎文件：`backend/app/detector.py`

主要能力：

1. 后端可选：`onnxruntime` 或 `opencv_dnn`
2. 预处理：等比缩放 + 填充（letterbox）
3. 后处理：阈值过滤 + NMS 去重 + 坐标回映射
4. 支持三种输出布局：
   - `cxcywh_obj_cls`
   - `xyxy_score_class`
   - `cxcywh_score_class`

## 3. API 设计

后端入口：`backend/app/main.py`

接口：

1. `GET /health`
2. `POST /api/model/load`
3. `POST /api/model/load_profile`
4. `POST /api/detect`
5. `POST /api/detect/batch`
6. `POST /api/report/export_csv`

## 4. 前端中文 GUI

前端入口：`frontend/lib/main.dart`

主工作台：`frontend/lib/pages/workbench_page.dart`

界面功能：

1. 后端连接测试
2. 模型文件选择
3. 单图检测
4. 批量检测
5. 阈值调节（置信度、IOU）
6. 缺陷统计与运行日志

## 5. 可扩展点

2. 增加 Word/Excel 报告模板导出
